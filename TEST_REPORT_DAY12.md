## 🎯 根本问题分析

### 核心问题：算法精度不足

#### 问题 1: FAQMatcher 相似度计算

**现状**:
- 用余弦相似度算法
- 需要 ≥ 0.9 置信度才能返回答案
- 中文分词词库：100+ 词

**为什么失败**:
```
输入：  "我的订单还没有到，已经超过了预计时间"
目标：  "我的订单什么时候到达？"

词汇分析：
- 输入词：订单、已经、超过、预计、时间
- FAQ 词：订单、时候、到达
- 共同词：订单 (1个)
- 余弦相似度：≈ 0.45 (远低于 0.9 阈值)
```

**解决方案**:
1. 📊 **降低置信度阈值** (从 0.9 → 0.75)
   - 权衡：降低精准度但提高覆盖率
   - 预期效果：FAQ 匹配率从 0% → 60%

2. 🔤 **扩展同义词库**
   - 添加更多相关词汇：到达→到了、已到、送到
   - 预期效果：提高相似度 +0.15-0.25

3. 🧠 **改进分词算法**
   - 使用更智能的中文处理
   - 预期效果：提高精准度

#### 问题 2: SimpleFilter 关键词匹配

**现状**:
- 关键词匹配计算：matched_keywords / total_keywords
- 需要 0.7-0.9 范围的置信度
- 每个分类 30-40 个关键词

**为什么失败**:
```
输入：  "物流"
分类：  shipping (30 个关键词)

关键词匹配：
- 输入词：物流 (1个)
- 分类关键词：物流、快递、配送... (30个)
- 置信度：1/30 = 0.033 (远低于 0.7 阈值)
```

**解决方案**:
1. 📝 **调整置信度计算**
   - 当前：matched / total
   - 建议：使用权重制（高频词权重更高）
   - 或降低最小阈值 (0.7 → 0.3)

2. 🎯 **优化关键词匹配**
   - 支持部分词汇匹配（如"物"和"流"分开也算匹配）
   - 预期效果：提高覆盖率
